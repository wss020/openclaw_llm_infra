# 🔄 模型升级与运维策略

本系统支持零停机升级。

---

# 一、升级流程

## 步骤

1. 修改 rewriteModel
2. 重启 Router
3. 验证健康接口
4. 回滚预案准备

---

# 二、升级示例

原配置：

```
rewriteModel: Qwen3-30B
```

升级为：

```
rewriteModel: Qwen3-32B
```

执行：

```
docker compose up -d --force-recreate model-router
```

---

# 三、回滚策略

保留旧模型路由：

```
openai/gpt-4o-old
```

快速切换虚拟模型名即可回滚。

---

# 四、灰度发布

可以：

- 为少量用户分配新模型
- 使用不同虚拟模型名测试
- 比较效果

---

# 五、升级原则

- 应用层永远不变
- Router 控制真实模型版本
- 必须支持快速回滚
- 记录每次模型变更日志

---

# 六、长期运维建议

- 定期测试健康接口
- 监控模型延迟
- 监控 Token 使用量
- 统计模型成本
- 自动告警机制
